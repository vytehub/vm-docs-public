Módulo de Offerings

Overview​

El módulo de Offerings permite a los usuarios crear y gestionar múltiples servicios publicables.
Un Offering representa un servicio que el usuario desea brindar o poner a disposición para que otros usuarios reserven.
Cada Offering se representa como Slots separados en el Timeline.

Valor clave:

  • Para Usuarios: configurar fácilmente su disponibilidad y compartir servicios reservables.
  • Para Otros: descubrir y reservar slots sin fricción, respetando reglas de disponibilidad.

Propiedades de un Offering​

Cada Offering cuenta con los siguientes atributos:

PropiedadDescripción
TitleTítulo que será mostrado en el timeline para identificar el Offering.
AvailabilityConjunto de parámetros que VyteMerge utiliza para calcular la disponibilidad (slots).
ExceptionsEl usaurio puede craer excepciones, creando configuraciones epecificas que no sigan el patron establecido para dias epecificos. por ejemplo agregar feriados. o dias especiales.
TimezoneZona horaria principal aplicada a este Offering.
Booking RulesReglas que definen el comportamiento y restricciones de las reservas para cada slot.
MediaImagen o short video. opcional alusiva al servicio/cita.
Availability Sync RulesEl usuario puede Agregar multiples reglas para sincronizar la disponibilidad con diferentes fuentes como otros timelines o offerings.
LocationSelección de un lugar en Google Maps (presencial) o Google Meet (online). Futuro: otros proveedores.
DescriptionTexto libre para describir el Offering.
PricePrecio del servicio (monto y moneda) que deberá pagar el usuario al reservar (si aplica).
Options and AdditionalsOpciones y adicionales a agregar a la offering que pueden modificar el precio y el servicio.
Intake FormFormulario dinámico configurado por el dueño; el usuario que reserva debe completarlo.
Tagstags que se utilizaran para filtrar los slots y bookings
RemindersRecordatorios configurables que se enviarán automáticamente a los usuarios que reservaron.
DisclaimerOpcionalmente el usuario podra agregar un disclaimer que sera exhibido al usuario reservante.

Availability​

El Availability define cómo se generan dinamicamente los slots de un Offering. Existen tres modos:

Configuración de Availability​

El usuario debe definir :

  • Duración de cada slot (ejemplo: 15m, 30m, 45m, 1h, 1h30m, 2h, o valor personalizado). Por defecto: 1h.
  • Margen entre slots (opcional; expresado en minutos u horas). por defecto estara deshabilitado.

👉 El sistema generará todos los slots posibles dentro de los rangos configurados, respetando la duración y los márgenes.

1. No se repite (Configuración Manual)​

  1. Selecciona los rangos horarios dentro de los cuales se generarán los slots de disponibilidad. Ejemplo: 09:00–13:30, 16:00–20:00.
  2. Indica uno o varios días específicos a los que se aplicará el rango configurado.
  3. Puede crear múltiples rangos de disponibilidad, con la restricción de que no es posible definir más de un conjunto de rangos para el mismo día.
  • El usuario configura manualmente la disponibilidad día por día.
  • Ejemplo: 
    22/08/2025, 23/08/2025 → 09:00–13:30 / 16:00–20:00  
    24/08/2025 → 09:00–11:30
  • Slots generados: 
    22/08/2025 → 09:00–10:00, 10:30–11:30, 12:00–13:00, 16:00–17:00, 17:30–18:30, 19:00–20:00  
    23/08/2025 → igual que 22/08/2025  
    24/08/2025 → 09:00–10:00, 10:30–11:30

2. Se repite semanalmente (Configuración Calculada)​

  1. Selecciona los rangos horarios dentro de los cuales se generarán los slots de disponibilidad. Ejemplo: 09:00–13:30, 16:00–20:00.
  2. Indica uno o varios días de la semana a los que se aplicarán los rangos. Opciones: Lun, Mar, Mie, Jue, Vie, Sab, Dom.
  3. Puede crear diferentes rangos de disponibilidad, con la restricción de que no es posible definir más de un conjunto de rangos para el mismo día de la semana.
  4. Define una fecha de inicio y una fecha de fin para aplicar el Offering. Si no se establece fecha de fin, el patrón se repetirá indefinidamente.
  • El usuario configura slots por día de la semana.
  • El mismo patrón semanal se repite indefinidamente (o hasta una fecha de fin).
  • Ejemplo: 
    Fecha de inicio: 22/08/2025  
    Fecha de fin: 25/08/2025  

    Lun, Mar → 09:00–13:30 / 16:00–20:00  
    Vie → 09:00–11:30
  • Slots generados: 
    Vie 22/08/2025 → 09:00–10:00, 10:30–11:30  
    Lun 25/08/2025 → 09:00–10:00, 10:30–11:30, 12:00–13:00, 16:00–17:00, 17:30–18:30, 19:00–20:00

3. Repetición Personalizada (Configuración Calculada)​

  1. Selecciona los rangos horarios dentro de los cuales se generarán los slots de disponibilidad. Ejemplo: 09:00–13:30, 16:00–20:00.
  2. Indica uno o varios días de la semana en los que se aplicarán los rangos. Opciones: Lun, Mar, Mié, Jue, Vie, Sáb, Dom.
  3. Puede crear diferentes rangos de disponibilidad, con la restricción de que no es posible definir más de un conjunto de rangos para el mismo día de la semana.
  4. Define una fecha de inicio y una fecha de fin para aplicar el Offering. Si no se establece fecha de fin, el patrón se repetirá indefinidamente.
  5. IMPORTANT! Establece cada cuántas semanas se repetirá la configuración (por ejemplo: cada 2 semanas, cada 3 semanas).
  • Similar a la repetición semanal, pero con una frecuencia personalizada (ej. cada 2 semanas).
  • Ejemplo (cada 2 semanas): 
    Fecha de inicio: 22/08/2025  
    Fecha de fin: 13/09/2025  

    Vie → 09:00–11:30
  • Slots generados: 
    Vie 22/08/2025 → 09:00–10:00, 10:30–11:30  
    Vie 05/09/2025 → 09:00–10:00, 10:30–11:30

Exceptions​

Las excepciones son configuraciones especiales que permiten al usuario agregar casos particulares en los que la disponibilidad se comportara de manera diferente.

  • Las excepciones esta disponibles solo para la Disponibilidad Calculada.
  • El usuario debera seleccionar un dia en el calendario para crear la excepcion.
  • Si la disponibilidad tiene fecha de inicio y fin, el dia a seleccionar debera estar dentro de ese rango.
  • El usuario podra escojer marcar el dia compo no disponible.
  • El usuario podra escoger ragos horarios en los cuales VyteMerge intentara crear slots de disponibilidad.

TimeZone​

El usuario puede configurar el timezone que se aplicara al offering. Esto permitira al usuario crear diferentes offerings en diferentes zonas horarias en el caso de brindar servicios en diferentes paises por ejemplo. por defecto se utilizara la zona horaria del navegador.

Booking Rules​

Se pueden configurar diferentes reglas que determinaran el comportamiento del offering al momento de la reserva.

  • Antelacion en dias: el usuario puede habilitar esta opcion para forzar que los slots solo puedan ser reservados con el tiempo en dias requerido. por defecto esta opcion esta deshabilitada.
  • Antelacion en horas: el usuario puede habilitar esta opcion para forzar que los slots solo puedan ser reservados con el tiempo en horas requerido. por defecto esta opcion esta deshabilitada.
  • Permiso para agregar personas: el usuario puede habilitar esta opcion para permitir a los usarios que reservan un slot invitar a mas personas al evento.
  • Maximo de reservas por dia: el usuario puede habilitar esta opcion para limitar la cantidad de bookings por dia para este offering.
  • Cupo de reservas por slot: el usuario puede habilitar esta opcion para cabiar el cupo bookings por cadamismo slot. Por defecto esta opcion sera 1. Esta opcion pretende permitir al usuario tener multiples bookings en el mismo slot, por ejemplo para una clase o webinar.
  • Requiere Confirmacion: Seleccionando esta opcion, la cita debera ser confirmada manualmente por el dueño del slot antes de figurar como pendiente. por defecto esta opcion estara deshabilitada.

Media​

El usuario podra escoger una o multiples imagenes o videos corto disponibles en su perfil o bien agregar nuevos.

IMPORTANTE Es necesario investigar sobre las restricciones en cuanto a la media queality y las restricciones tecnicas.

Availability Sync rules​

El usuario podra agregar multiples rules para syncronizar y limitar su disponibilidad y evitar overlays indeseados.

  • Source: El usuario podra seleccioanar como fuente de sincronizacion las siguientes opciones

    1. Timelines propios
    2. Timelines compartidos por otros usuarios
    3. Offerings dentro del mismo time line.

  • Filter:

    1. Time line filters: El usuario podra agregar multiples tags y offerings a ser consultados.
    2. Status: Agregar el estado de los bookings y eventos a consultar, por ej: Pendiente, Confirmado. etc.

  • Slots Action on match: El usuario podra elegir las sigueintes opciones

    1. Hide availability slots
    2. Notify me, I will decide.

  • Events and Bookings Action on match: En el caso en que un evento sea agregado en un time line asociado y existan un evento o booking en el time line actual.

    1. Notify me, I will decide.
    2. Cancel Event automatically and try to reschedule.
    3. Cancel Event automatically and not try to reschedule.

Location​

El usuario podra escoger la forma y el lugar en el que se llevara a cabo el evento una vez echo bocked del slot. Estas seran las opciones.

  • My Locations: Se podra escoger o crear una Location. esta location tendra su propia info, ubicacion, fotos etc. por ej: Mi consultorio. (esto se profundizara en otro documento).
  • Face to face meeting: On this case the user must pick a location on the map (is required).
  • Phone Call: Al momento de hacer el book the slot el usuario que reserva debera proporcionar un numero de telefono requerido.
  • Video call: Esta opcion en una primera instancia generara un link de google meet automaticamente para esta opcion.
  • None: Permitira al usuario decidir en un futuro y en comun acuerdo con el usuario solicitante la manera de llevar a cabo el evento.

Nota: El usuario podra seleccionar mas de un tipo de location para ofrecer a sus usarios reservantes. por ejemplo, el usuario reservante podra escoger entre una video llamada o una cita precencial para llevar a cabo la cita.

Description​

Esta opcion permitira al usuario agregar algun detalle respecto al offering, debera ser texto enriquecido por lo que se evaluara algun pluguin o tool para permitir el formato del mismo.

Price​

El usuario podra establecer un precio y un currency del offering. Ademas del precio del offering el usuario podra configurar las siguientes opciones para la realizacion del pago.

  1. Pay Total: El usuario reservante debera abonar la totalidad al momento de la reserva para poder agendar el evento.
  2. Pay percentage: El usuario debera pagar un porcentaje del total a modo de "seña" para agendar la reserva.
  3. Pay fixed: El usuario abonara un monto fijo establecido al momento de ralizar la reserva.
  4. Medios de pago: En una fase dos se podra utilizar una plataforma propia. en una primera instancia se podra optar por las siguientes opciones
    1. Efectivo: no se abona en la reserva si no al momento de ejecutar la cita.
    2. Transferencia: Se podra compartir un comprobante de transferencia. para realizar la reserva.
    3. Mercado Pago: esta sera la unica opcion en la primera instancia. le usuario podra vincular su cuenta de mercado pago para recibir el pago

Nota: Se puede seleccionar multiples medios de pago para poner a disposicion.

Options and Additionals​

En conjunto con el precio se podran establecer opciones y adicionales ambos podran tener un precio que cambiaran el precio total

  1. Options: Se podra crear o utilizar un grupo de opciones, cada opcion podra tener un precio adicional. por ej. En el caso de reservar un turno para el lavado del auto, podra tener opciones como ser, encerado, lavado profundo, pulido. entre otras opciones que haran variar el precion del servicio. Podemos ver a las opciones como variantes del mismo servicio.
  2. Adicionales: Los adicionales son servicios o productos que el usuario reservante podra añadir a su reserva si asi lo desea. los mismos tambien podran contrar o no con un precio definido que afectara al valor total del servicio.

Otro ejemplo de opciones puede ser

TURNO MEDICO: CON IPS TIENE UN PLUS DE 100. CON OSDE CERO, PARTICULAR 30.000...

Nota: SIEMPRE Y CUANDO SE ENTIENDA QUE EL TAMAÑO DEL SLOT NO SE ME MODIFICA. SI QUIERE TAMAÑOS DIFERENTES DE SLOTS DEBE AGREGAR MULTIPLES OFFERINGS.

Intake Form​

Al momento de reservar un slot el usuario reservante debera completar algunos campos requeridos que seran Name,LastName,Email. ademas al momento de configurar la offering se podran agregar campos adicionales como ser Phone u otros campos custom. En una primera instancia solamente se sportaran el tipo Text o Input.

Tags​

Se podran agregar multiples tags, existentes o nuevos, la idea de los tags es poder tener una forma de filtrar eventos, slots. etc. existiran tres tipos de tags. tags globales (pertenecientes a vyemerge), tags privados, estos solo seran visibles para el creador. y serviran solamente para filtro. y tags publicos propios. estos tags serviran para que los usuarios reservantes puedan filtrar los direntes offeringsen el profile del usuario.

Nota : Trabajar en terminar de definir como se comportaran y relacionaran los 3 tipos de tags.

Disclaimer​

El disclaimer un legal que el usuario podra agregar, para ser leido y aceptado por el usuario reservante al momento de hacer un book de un slot.

Criterios de Aceptación (MVP)​

  • El usuario puede crear múltiples Offerings, cada uno generando Slots en su Timeline.
  • Los Patterns soportan configuraciones Manual, Semanal y Personalizada.
  • Los Offerings respetan Zona horaria, Reglas de Agenda y Precio.
  • El Formulario de Reserva debe completarse antes de confirmar un slot.
  • Los Recordatorios se envían automáticamente según configuración.
  • Los Slots se generan automáticamente dentro de los rangos definidos, respetando duración y margen.
