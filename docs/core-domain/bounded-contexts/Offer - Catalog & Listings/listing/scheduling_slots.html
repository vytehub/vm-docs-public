Scheduling & Slots

TL;DR​

Un Service Listing puede ser reservable porque define un Scheduling Pattern (un patrón de horarios). A partir de ese patrón, el sistema genera Slots (huecos reservables) para mostrar y permitir bookings.

VyteMerge soporta 3 modos de scheduling:

  1. Semanal (se repite todas las semanas)
  2. Personalizado (se repite cada N semanas)
  3. Día por día (configuración manual por fecha)

Además, se soportan Excepciones para modificar días puntuales.

Los Slots son una proyección.
Cuando se crea un Booking, el sistema debe revalidar conflictos/capacidad antes de confirmar.

Conceptos base​

Slot Duration y Gap​

Para generar slots dentro de un rango horario se usan:

  • SlotDuration: duración del slot (ej: 15m, 30m, 60m, 90m)
  • GapBetweenSlots (opcional): margen entre slots (ej: 0m, 10m, 30m)

Regla simple​

Dado un rango [Start, End):

  • Se van creando slots consecutivos de SlotDuration
  • Entre slots se suma GapBetweenSlots
  • No se generan slots que excedan End

Ejemplo:

  • Rango: 09:00–11:30
  • SlotDuration: 01:00
  • GapBetweenSlots: 00:30

Slots:

  • 09:00–10:00
  • 10:30–11:30

Modo 1: Semanal​

Qué hace​

Define disponibilidad por día de la semana y la repite todas las semanas.

Configuración​

  • Seleccionar días: Lun..Dom
  • Para cada día, definir uno o más rangos (ej: 09:00–13:30 y 16:00–20:00)
  • Definir StartDate (desde cuándo aplica)
  • EndDate (opcional): hasta cuándo aplica

Reglas de consistencia​

  • Un mismo día de semana puede tener varios rangos, pero deben ser no solapados.
  • Si EndDate no existe, el patrón es indefinido (hasta que se cambie).

Modo 2: Personalizado (cada N semanas)​

Qué hace​

Igual que Semanal, pero el patrón se repite cada N semanas.

Configuración​

  • Días: Lun..Dom
  • Rangos por día (no solapados)
  • StartDate
  • EndDate (opcional)
  • RepeatEveryNWeeks (ej: 2, 3, 4)

Ejemplo​

  • RepeatEveryNWeeks = 2
  • Configuro viernes 09:00–11:30

Se generan slots solo en “viernes alternos” dentro del rango de fechas configurado.

Modo 3: Día por día (manual por fecha)​

Qué hace​

Define disponibilidad para fechas específicas, sin repetición automática.

Configuración​

  • Elegir fecha(s) en calendario
  • Definir rangos para cada fecha (ej: 09:00–12:00)
  • Definir SlotDuration y GapBetweenSlots

Reglas de consistencia​

  • Una misma fecha puede tener varios rangos, pero deben ser no solapados.
  • Este modo es ideal para:
    • agendas irregulares
    • semanas especiales
    • eventos únicos

Excepciones (Exceptions)​

Qué son​

Las excepciones permiten cambiar un día puntual aunque exista un patrón semanal o personalizado.

Ejemplos típicos:

  • feriado (cerrado)
  • día especial con horario distinto
  • bloqueo total por viaje/congreso

Tipos de excepción​

  • BlockDay: bloquear todo el día (sin slots)
  • OverrideRanges: reemplazar rangos del día por rangos específicos

Prioridad​

Para una fecha concreta:

  1. Si existe excepción → se aplica la excepción
  2. Si no existe excepción → se usa el patrón (semanal/personalizado/día por día)

Qué pasa cuando cambian reglas o se confirma un booking​

Revalidación al reservar​

Al confirmar un booking, el sistema debe revalidar:

  • Conflicts del Timeline (Conflict Rules)
  • Shared Capacity / Staff pools (si aplica)
  • Bloqueos (Access / Agreements si afecta visibilidad/acciones)

Esto evita overbooking si un slot quedó “viejo” porque cambió el contexto.

Cambios en scheduling​

Si el owner cambia el scheduling:

  • cambia la proyección de slots futuros
  • bookings existentes NO se borran (solo se afectan slots futuros)

Checklist​

  • Soportar 3 modos: Semanal, Personalizado cada N semanas, Día por día
  • Rangos no solapados por día/fecha
  • Excepciones con prioridad sobre el patrón
  • Revalidación al crear booking
