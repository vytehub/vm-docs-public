Tags

Identidad semántica, búsqueda y coordinación en VTWA​

1. Visión general​

El módulo Tags es un componente central de VTWA que provee un lenguaje común para:

  • Búsqueda global (marketplace)
  • Filtrado de Offerings y Timelines
  • Coordinación de disponibilidad
  • Reglas de agenda y capacity
  • Descubrimiento social

Un Tag representa un concepto semántico reutilizable en todo el sistema.

Ejemplos:

  • #medico
  • #traumatologo
  • #rodilla
  • #manicura
  • #bloqueo
  • #guardia

2. Principios de diseño​

2.1 Los tags son un sistema, no un string libre​

  • No se guardan strings sueltos
  • Todo se resuelve a un TagId
  • Permite consistencia, analytics y evolución futura

2.2 Un tag tiene identidad y contexto​

Un tag puede existir en distintos alcances (scope):

  • Global
  • De usuario
  • De timeline

2.3 Resolver > Crear​

  • El flujo principal es resolver un tag
  • Crear es una consecuencia controlada
  • Evita explosión de tags duplicados

3. Qué problemas resuelve​

Sin un sistema de tags:

  • Búsquedas inconsistentes
  • Filtros frágiles
  • Duplicación de conceptos
  • Reglas difíciles de explicar

Con Tags:

  • Búsqueda rica y flexible
  • Reglas explicables
  • Evolución hacia ML / recomendaciones
  • Base para una red social semántica

4. Qué NO hace el módulo Tags​

  • ❌ No aplica reglas de negocio
  • ❌ No interpreta disponibilidad
  • ❌ No decide conflictos
  • ❌ No asigna recursos

👉 Solo define y resuelve conceptos semánticos.

5. El modelo Tag​

Tag
 ├─ Id
 ├─ Slug
 ├─ DisplayName
 ├─ Scope
 ├─ OwnerId?
 ├─ Status
 ├─ Kind
 ├─ UsageCount
 ├─ CreatedAtUtc
 └─ UpdatedAtUtc

6. Slug vs DisplayName​

Slug​

  • Identificador técnico
  • Normalizado
  • Único por (Scope + OwnerId)
  • Usado para búsqueda y matching

DisplayName​

  • Texto visible al usuario
  • Editable
  • Puede cambiar sin romper referencias

7. Scope (alcance)​

Global​

Tags compartidos por todo el sistema (marketplace).

User​

Tags personales, privados.

Timeline​

Tags contextuales a un timeline específico.

8. ResolveTag (operación central)​

La operación principal del módulo es:

Resolver un input a un TagId

9. TagResolveContext​

Define la política de resolución según el caso de uso:

  • GlobalSearch
  • TimelineConfig
  • UserPersonal
  • OfferingConfig
